# Correlation-Aware Uncertainty Estimation

## Purpose

Monte Carlo measurements generated by a Markov chain are generally correlated.

This creates an important statistical problem. Ordinary bootstrap resampling of individual observations implicitly treats those observations as exchangeable independent measurements.

For strongly autocorrelated data, that approximation can produce misleading uncertainty estimates.

The Yang-Mills Mass Gap Laboratory therefore includes correlation-aware resampling methods.

## Markov-Chain Correlation

Suppose an observable is measured along a Markov chain:

X_1, X_2, ..., X_N.

The samples are generally not independent.

A configuration at one sweep is produced from the previous configuration, so nearby measurements may contain related information.

The integrated autocorrelation time provides a rough summary of this dependence.

The effective sample size is approximated by:

N_eff = N / (2 tau_int).

Therefore N recorded measurements may contain substantially less information than N independent samples.

## Naive Bootstrap

The naive bootstrap draws individual measurements independently with replacement.

This is useful as a baseline, but it can destroy temporal dependence.

The repository retains this method so its results can be compared with correlation-aware estimators.

## Circular Moving-Block Bootstrap

The moving-block bootstrap resamples contiguous groups of measurements.

For a block size b, the time series is treated as a collection of local segments:

X_s, X_(s+1), ..., X_(s+b-1).

The repository uses circular indexing, allowing a sampled block to wrap from the end of the time series back to the beginning.

Multiple sampled blocks are concatenated and truncated to the original sample length.

This preserves short-range ordering inside each block.

## Block Size

Block size is a critical modeling choice.

If the block size is too small, important autocorrelation may be destroyed.

If the block size is too large, the number of effectively distinct blocks becomes small and the uncertainty estimator can become noisy.

The project therefore performs block-size sensitivity analysis.

The experiment compares standard-error estimates across several block sizes.

A heuristic candidate block size is constructed from approximately twice the estimated integrated autocorrelation time.

This is an exploratory diagnostic rather than a universal optimality rule.

## Delete-One Jackknife

For N measurements, the ordinary delete-one jackknife constructs N replicate estimates.

Each replicate removes one observation.

The variability of these replicate estimates is used to estimate standard error.

## Delete-One-Block Jackknife

For correlated data, the repository also implements a block jackknife.

The time series is divided into complete non-overlapping blocks.

Each replicate removes one entire block rather than one individual measurement.

This allows the estimator to respond to dependence occurring within blocks.

## Why This Matters

Earlier versions of the project used ordinary bootstrap uncertainty estimates.

The correlation-aware resampling layer now makes a more important distinction:

raw number of Monte Carlo measurements is not the same as number of independent measurements.

The project can now compare:

- naive bootstrap uncertainty,
- moving-block bootstrap uncertainty,
- delete-one jackknife uncertainty,
- delete-one-block jackknife uncertainty,
- effective sample size,
- block-size sensitivity.

## Scientific Limitation

No resampling method automatically converts a small finite-lattice simulation into a rigorous Yang-Mills result.

The quality of an uncertainty estimate still depends on:

- chain length,
- thermalization,
- observable choice,
- autocorrelation structure,
- block-size selection,
- number of independent ensembles,
- numerical stability.

These methods improve the statistical methodology of the finite-lattice study. They do not prove the Yang-Mills mass gap.
